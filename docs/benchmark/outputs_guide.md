## 3) Field Meaning By Directory

### `RUN_SUMMARY.md`

Auto-generated short overview:

- checkpoint used
- selected cases
- quick aggregate metrics
- first files to inspect

Use this first when opening an unfamiliar run.

### `config_snapshot/`

Reproducibility snapshot:

- `env_base.yaml`: environment config before case overrides.
- `agent_base.yaml`: runner/agent config.
- `runtime_args.json`: actual CLI args used.
- `benchmark_cfg_resolved.json`: parsed benchmark config content.
- `selected_cases.json`: explicit case list for this run.

If you want to **re-run exactly**, start here.

### `raw_metrics/<suite>/<case>/`

Case-level raw data:

- `episodes.csv` / `episodes.json`:
  - one row per completed episode.
  - includes return, episode length, tracking errors, fall/timeout, termination reason, recovery counters, stability proxies.
- `summary.json`:
  - single-case aggregate produced by the runner.
- `termination_stats.csv`:
  - termination reason counts.
- `case_overrides_snapshot.json`:
  - exact overrides used for this case.

If you want to answer “why did this case fail?”, inspect this folder first.

### `traces/<suite>/<case>/trace_env.csv`

Step-level time series (single traced env):

- command vx/vy/wz
- actual vx/vy/wz
- base height
- roll/pitch/yaw

Use for behavior diagnosis and recovery analysis.

### `videos/<suite>/<case>/`

Representative clips from that case.
Current naming from Gym `RecordVideo`:

- `rl-video-step-20.mp4` (step number depends on trigger)

### `summary_metrics/`

Cross-case aggregates for this run:

- `case_summary.csv` / `case_summary.json`:
  - one row per case.
- `index.json`:
  - run manifest:
    - run_dir, checkpoint, config path, case count, elapsed time
    - pointers to summary and report artifacts

### `plots/`

Auto-generated figures from `analysis.py`:

- `case_bar_mean_return.png`
- `case_bar_mean_episode_length.png`
- `case_bar_fall_rate.png`
- `case_bar_mean_lin_vel_tracking_error.png`
- bucket-level bar charts
- optional `combined_ood_fall_rate_heatmap.png`

### `reports/`

- `research_summary.md`: auto summary text.
- `manifest.json`: list of generated report artifacts.

## 4) Isolated Master Layout

Generated by `run_quadruped_benchmark_isolated.py`.

Real example:

- [outputs/quadruped_rough_benchmark/full_video_representative_2026-04-30](../../outputs/quadruped_rough_benchmark/full_video_representative_2026-04-30)

Structure:

```text
<master_dir>/
  RUN_SUMMARY.md
  manifest.json
  selected_cases.json
  runs/
    01_<suite>__<case>/
      ... (full per-run structure from structured runner)
    02_<suite>__<case>/
      ...
  summary_metrics/
    case_summary.csv
    case_summary.json
    failures.json
  plots/
    case_bar_*.png
    bucket_bar_*.png
  reports/
    research_summary.md
    manifest.json
```

Naming rule for per-case run directories:

- `<index:02d>_<suite_name>__<case_name>`
- Example: `05_observation_mismatch__obs_ood_delay_drop`

Interpretation:

- `05`: execution order in isolated batch
- `observation_mismatch`: suite bucket family
- `obs_ood_delay_drop`: case identifier

## 7) What To Read First For Common Questions

Question: “What did this run evaluate?”

- open `<run_dir>/RUN_SUMMARY.md`
- then `<run_dir>/config_snapshot/selected_cases.json`

Question: “Where are final comparable numbers?”

- `<run_dir>/summary_metrics/case_summary.csv`

Question: “Where is exact case config?”

- `<run_dir>/raw_metrics/<suite>/<case>/case_overrides_snapshot.json`

Question: “Where are failure examples?”

- `<run_dir>/videos/<suite>/<case>/`
- `<run_dir>/raw_metrics/<suite>/<case>/episodes.csv` (`termination_reason`, `fell`)

Question: “How do I compare two runs?”

- compare each run’s `summary_metrics/case_summary.csv`
- if merged, use merged run’s summary + plots + report

## 8) Naming Semantics Summary

- `suite_name`: benchmark factor family (e.g. `dynamics_mismatch`)
- `case_name`: concrete condition (e.g. `dynamics_ood_extreme`)
- `bucket`: semantic severity label (`id`, `long_tail`, `ood`)
- `grid_x/grid_y` (when present): metadata for combined-OOD heatmap positioning
